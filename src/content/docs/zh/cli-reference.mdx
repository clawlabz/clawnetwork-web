## 概述

ClawNetwork 提供两个 CLI 工具：

- **`claw-node`** — 全节点，用于同步、验证和质押
- **`claw-miner`** — 挖矿客户端，通过贡献资源赚取 CLAW

两者均可从 [GitHub Releases](https://github.com/clawlabz/claw-network/releases) 页面下载预编译二进制文件，或使用[快速开始](/docs/quickstart)指南中的一键安装脚本。

### 全局参数

以下参数适用于所有 `claw-node` 子命令：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--data-dir` | `~/.clawnetwork` | 数据目录路径 |
| `--log-format` | `text` | 日志输出格式：`text` 或 `json` |

## claw-node

### `claw-node init`

初始化新的节点数据目录，生成密钥文件和配置文件。

```bash
claw-node init [OPTIONS]
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--network` | `devnet` | 加入的网络：`devnet`、`testnet`、`mainnet` |

**示例：**

```bash
# 为主网初始化
claw-node init --network mainnet

# 为测试网初始化，使用自定义数据目录
claw-node init --network testnet --data-dir /opt/claw-data
```

创建以下目录结构：

```
~/.clawnetwork/
├── config.toml          # 节点配置
├── key.json             # Ed25519 密钥文件（请妥善保管！）
└── data/                # 链数据（区块、状态）
```

### `claw-node start`

启动节点，开始与网络同步。

```bash
claw-node start [OPTIONS]
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--network`、`-n` | （来自配置） | 连接的网络 |
| `--rpc-port` | （来自配置） | JSON-RPC 服务端口 |
| `--p2p-port` | （来自配置） | P2P 网络端口 |
| `--bootstrap` | （内置） | 额外的引导节点地址（multiaddr 格式） |
| `--single` | `false` | 强制单节点模式（无 P2P） |
| `--sync-mode` | `full` | 同步模式：`full`、`fast` 或 `light` |
| `--allow-genesis` | （网络预设） | 当没有链数据时允许创建新的创世块。默认值：devnet 为 `true`，mainnet/testnet 为 `false` |
| `--skip-version-check` | `false` | 跳过启动时的版本检查（仅用于开发/紧急情况） |

**示例：**

```bash
# 使用默认配置启动（主网）
claw-node start --network mainnet

# 自定义 RPC 端口
claw-node start --network mainnet --rpc-port 8545

# 单节点模式启动（无 P2P，用于开发）
claw-node start --network devnet --single

# 使用自定义引导节点
claw-node start --network mainnet --bootstrap /ip4/178.156.162.162/tcp/9711
```

节点将：
1. 从数据目录加载配置和密钥文件
2. 通过 P2P 连接到引导节点
3. 从网络同步区块
4. 启动 JSON-RPC 服务器

### `claw-node status`

打印 RPC 端点 URL，用于查询节点状态。

```bash
claw-node status
```

**示例：**

```bash
claw-node status

# 输出：
# claw-node status: use RPC at http://localhost:9710
```

使用 JSON-RPC 端点查询详细的节点信息，如区块高度、对等节点数和同步状态。

### `claw-node stake`

质押 CLAW 代币成为验证者。采用单一所有者委托模型。

```bash
claw-node stake <AMOUNT> [OPTIONS]
```

| 参数 / 标志 | 说明 |
|-------------|------|
| `<AMOUNT>` | 质押的 CLAW 数量（整数单位） |
| `--validator-key` | 委托给指定验证者地址（十六进制，64 字符）。省略则为自质押。 |
| `--commission` | 佣金率（基点，0-10000，默认：`8000`） |
| `--rpc` | RPC 端点（默认：`http://localhost:9710`） |

**示例：**

```bash
# 自质押 10,000 CLAW
claw-node stake 10000 --rpc https://rpc.clawlabz.xyz

# 委托给验证者，佣金率 80%
claw-node stake 50000 --validator-key 3f2a...b817 --commission 8000
```

最低质押金额：

| 网络 | 最低质押 |
|------|---------|
| 主网 | 10,000 CLAW |
| 测试网 | 100 CLAW |

### `claw-node unstake`

解除质押 CLAW 代币。需等待解绑期。

```bash
claw-node unstake <AMOUNT> [OPTIONS]
```

| 参数 / 标志 | 说明 |
|-------------|------|
| `<AMOUNT>` | 解除质押的 CLAW 数量 |
| `--validator-key` | 解除质押的验证者地址（作为质押所有者解除质押时必填） |
| `--rpc` | RPC 端点（默认：`http://localhost:9710`） |

**示例：**

```bash
claw-node unstake 50000 --rpc https://rpc.clawlabz.xyz
```

主网解绑期为 **7 天**。在此期间，代币保持锁定状态且不产生奖励。

```bash

```

| 参数 | 说明 |
|------|------|
| `--rpc` | RPC 端点（默认：`http://localhost:9710`） |

**示例：**

```bash

```

这是一个免 gas 的交易。应每约 1000 个区块（约 50 分钟）发送一次。未发送信号超过 2000 个区块（约 100 分钟）后将被标记为不活跃。

## claw-miner

### `claw-miner init`

初始化矿工钱包和配置。

```bash
claw-miner init [OPTIONS]
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--data-dir` | `~/.clawminer` | 矿工数据目录 |
| `--rpc` | `https://rpc.clawlabz.xyz` | 连接的节点 RPC 端点 |

**示例：**

```bash
# 使用默认配置初始化
claw-miner init

# 使用自定义 RPC 初始化
claw-miner init --rpc http://localhost:9710

# 使用自定义数据目录初始化
claw-miner init --data-dir /opt/claw-miner
```

创建：

```
~/.clawminer/
├── config.toml     # 矿工配置
├── key.json        # 矿工钱包密钥文件
└── models/         # 本地模型存储
```

---

### `claw-miner start`

启动挖矿进程。矿工将向网络注册并开始赚取奖励。

```bash
claw-miner start [OPTIONS]
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--tier` | `1` | 挖矿层级：`1`（Standard）— 第一阶段仅支持 Tier 1 |
| `--rpc` | （来自配置） | 节点 RPC 端点 |
| `--data-dir` | `~/.clawminer` | 矿工数据目录 |
| `--log-level` | `info` | 日志详细程度 |

**示例：**

```bash
# 启动 Tier 1 挖矿（Standard）
claw-miner start

# 使用自定义数据目录启动 Tier 1
claw-miner start --data-dir /opt/claw-miner
```

Tier 1 的要求和奖励详情请参阅 [Agent 挖矿](/docs/mining)。**Phase 1 仅支持 Tier 1（在线/标准挖矿）。** Tier 2 和 Tier 3 将在后续阶段推出。

---

### `claw-miner stop`

优雅地停止挖矿进程。

```bash
claw-miner stop [OPTIONS]
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--data-dir` | `~/.clawminer` | 矿工数据目录 |

**示例：**

```bash
claw-miner stop

# 使用自定义数据目录停止
claw-miner stop --data-dir /opt/claw-miner
```

矿工在关闭前会从网络注销。强制终止进程可能会暂时降低您的信誉分。

| `--model` | 指定共享的模型（默认：提供商的默认模型） |
| `--endpoint` | 自定义 API 端点（用于自托管或 Ollama） |

**示例：**

```bash
# 共享您的 DeepSeek 订阅
claw-miner llm add deepseek --api-key sk-xxx

# 共享本地 Ollama 模型
claw-miner llm add ollama --endpoint http://localhost:11434 --model llama3.1

# 共享 OpenAI 访问权限
claw-miner llm add openai --api-key sk-xxx --model gpt-4o-mini
```

您的 API 密钥存储在本地，永远不会发送到网络。只有推理请求和响应通过系统流转。

## 环境变量

两个工具均支持以下环境变量：

| 变量 | 说明 |
|------|------|
| `CLAW_DATA_DIR` | 覆盖默认数据目录 |
| `CLAW_RPC_URL` | 覆盖默认 RPC 端点 |
| `CLAW_KEY_PASSWORD` | 加密/解密 key.json 的密码 |
| `CLAW_LOG_LEVEL` | 设置日志详细程度（`error`、`warn`、`info`、`debug`、`trace`） |
| `CLAW_LOG_FILE` | 将日志写入文件而非 stderr |

当同时提供环境变量和 CLI 参数时，CLI 参数优先。
