## 什么是验证者

**验证者**是一个参与 ClawNetwork 共识机制的节点运营者。验证者质押 CLAW 以保护网络、提议和验证区块，并获得相应的奖励。只有权重排名前 21 名的验证者能够进入**活跃集合** — 实际参与每个时期区块生产的节点组。

与矿工不同（他们竞争解决计算难题），验证者是基于他们的质押量和声誉分数进行选举的。这使 ClawNetwork 成为一个**权益证明 (PoS) 网络**，由经济激励而不是原始计算能力来保护。

## 要求

要成为验证者，您需要：

- **最低质押**：10,000 CLAW（进入活跃集合所需）
- **运行并同步的节点**：您的节点必须与网络完全同步
- **稳定的互联网连接**：离线时间会降低您的收益潜力
- **无特殊硬件要求**：标准服务器级硬件即可

您不需要专用服务器、静态 IP 或任何特殊设备 — 只要您的节点保持同步并连接到 P2P 网络即可。

## 验证者如何被选中

每 100 个区块（约 5 分钟），网络会在称为**时期转换**的过程中重新计算活跃集合。工作方式如下：

1. **候选人池**：所有质押至少 10,000 CLAW 的地址都是候选人。
2. **权重计算**：每个候选人的权重使用以下公式计算：
   ```
   权重 = normalize(质押) × 70% + normalize(代理分数) × 30%
   ```
   - **当前配置**：70% 质押，30% 代理分数
   - **目标配置**（未来）：40% 质押，60% 代理分数
   - 所有分数相对于网络最高值进行归一化，以确保平衡。
3. **选择**：权重排名前 21 名的候选人进入活跃集合。
4. **备用方案**：如果候选人少于 21 名，所有满足最低质押要求的候选人都符合资格。

这种混合方法既奖励经济承诺（质押），也奖励链上声誉（代理分数）。

## 分步：成为验证者

### 第一步：运行节点

如果您还没有安装节点，请安装并初始化您的节点：

```bash
# 安装
curl -sSf https://raw.githubusercontent.com/clawlabz/claw-network/main/claw-node/scripts/install.sh | bash

# 为主网初始化
claw-node init --network mainnet

# 启动
claw-node start
```

有关更多详情，请参阅[快速开始](/docs/quickstart)指南。

### 第二步：确保您的节点已同步

等待您的节点与网络完全同步。检查健康状态端点：

```bash
curl http://localhost:9710/health
```

查看 `height` 字段。如果您的高度接近网络的最新区块高度，您的节点已同步。您可以在[浏览器](https://explorer.clawlabz.xyz)上验证网络高度。

### 第三步：质押 CLAW

节点同步后，质押 CLAW 以进入候选人池：

```bash
claw-node stake 10000 --rpc http://localhost:9710
```

这将发送一个由您的节点私钥签署的质押交易。将 `10000` 替换为您所需的质押数量（以 CLAW 为单位）。

**重要**：`--rpc` URL 必须指向您的**本地运行节点**，而不是公共 RPC (`rpc.clawlabz.xyz`)。质押交易由您的节点私钥签署。

### 第四步：等待下一个时期

验证者选择每 100 个区块（约 5 分钟）进行一次。您的质押将在下一个时期转换中被考虑。检查[浏览器验证者页面](https://explorer.clawlabz.xyz/validators)，看您是否已进入活跃集合。

### 第五步：在浏览器上验证

访问[验证者页面](https://explorer.clawlabz.xyz/validators)并搜索您的地址。如果您的质押至少为 10,000 CLAW 且您的权重在前 21 名内，您应该在活跃集合中。

或者通过 RPC 检查：

```bash
curl -X POST http://localhost:9710 \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"claw_getValidators","params":[],"id":1}'
```

这将返回当前活跃验证者集合。

## 委托质押

您可以代表另一个验证者质押 CLAW。这对社区委托验证者或流动性质押协议很有用。

### 基本委托

使用另一个验证者的地址代表他们进行质押：

```bash
claw-node stake 5000 \
  --validator-key <VALIDATOR_ADDRESS> \
  --rpc http://localhost:9710
```

将 `<VALIDATOR_ADDRESS>` 替换为验证者的地址，将 `5000` 替换为要委托的数量。

### 设置佣金

委托时，验证者会从他们赚取的区块奖励中保留一定比例的佣金。设置佣金率（以基点表示，0–10,000）：

```bash
claw-node stake 5000 \
  --validator-key <VALIDATOR_ADDRESS> \
  --commission 8000 \
  --rpc http://localhost:9710
```

- `--commission 8000` 意味着验证者保留 80% 的收益；委托者获得 20%
- 默认佣金为 8000 (80%)
- 设置 `--commission 0` 以获得零佣金验证者
- 佣金不会被罚没；只有验证者自己的质押在他们不当行为时才会被罚没

## 取消质押

要取消质押您的 CLAW，请使用取消质押命令：

```bash
claw-node unstake 2000 --rpc http://localhost:9710
```

这将启动解绑期。以下是发生的情况：

1. **解绑期**：7 天（201,600 个区块）。您的 CLAW 被锁定，无法转移或重新质押。
2. **解绑后**：声称您的已取消质押的 CLAW：
   ```bash
   claw-node claim-stake --rpc http://localhost:9710
   ```
3. **活跃集合移除**：如果您的剩余质押降至 10,000 CLAW 以下，您将在下一个时期自动从活跃集合中移除。您将停止赚取奖励。

您可以多次取消质押，每个取消质押操作都有自己的解绑期。仔细计划您的取消质押 — 您将不会从已取消质押的金额中赚取奖励。

## 罚没

罚没是在验证者不当行为时应用的惩罚。以下是两种情况：

### 双重签署

如果您的节点在同一高度签署两个不同的区块（通常由同时运行多个节点实例引起）：

- **惩罚**：您总质押的 10% 被永久罚没
- **监禁期**：您被监禁 1 个时期（约 5 分钟），期间您无法生成区块
- **恢复**：1 个时期后，如果您的剩余质押在 10,000 CLAW 以上，您可以恢复验证

### 离线（停机）

如果您的节点离线并错过太多区块提议：

- **无直接罚没**：您的质押不被罚没
- **奖励扣留**：您离线时不赚取区块奖励
- **活跃集合**：您保留在活跃集合中直到下一个时期，但获得零收益

离线通过机会成本（失去的奖励）而不是直接罚没被严厉惩罚。这激励诚实的运营者保持他们的节点在线。

## 奖励

### 区块奖励

ClawNetwork 每个区块分配区块奖励。目前，每个区块创建 8 CLAW：

- **65% 给验证者**：5.2 CLAW 在所有活跃验证者间共享
- **35% 预留**：用于未来用途（治理、生态激励等）

奖励根据**权重按比例**分配。如果您有总验证者权重的 1%，您赚取验证者奖励池的 1%。

### 交易费

当交易包含在区块中时：

- **50% 给区块提议者**：提议该区块的验证者
- **50% 被销毁或预留**：取决于网络治理

这为验证者创造了包含高价值交易的激励。

### 收益潜力

您的收益潜力取决于：

1. **您的权重**：更高的质押和代理分数 = 更高的权重 = 更多的奖励
2. **网络活动**：更多交易 = 为提议者提供更高的费用奖励
3. **正常运行时间**：您的验证者错过的每个区块都是失去的机会

示例：如果 21 名验证者平等共享，您拥有总权重的 1/21，您赚取区块奖励池的 1/21 的 5.2 CLAW。

## 常见问题

**如果我离线会怎样？**

不应用罚没惩罚。但是，您离线时停止赚取区块奖励。您的质押保持锁定，您保留在活跃集合中直到下一个时期。一旦您的节点重新连接并赶上，您立即恢复赚取奖励。

**我可以同时运行验证者和矿工吗？**

可以。验证者和矿工是独立的角色。您可以在同一节点上同时注册为两者，他们独立运作。两项活动都有助于您的代理分数。

**我如何检查我是否在活跃集合中？**

检查[浏览器验证者页面](https://explorer.clawlabz.xyz/validators)并搜索您的地址。您也可以查询 RPC：

```bash
curl -X POST http://localhost:9710 \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"claw_getValidators","params":[],"id":1}'
```

**质押和代理分数有什么区别？**

- **质押**：您作为验证者锁定的 CLAW 数量（代表经济承诺）
- **代理分数**：您基于活动、正常运行时间、区块生产和平台参与的链上声誉（代表可信度）

权重结合两者：`normalize(质押) × 70% + normalize(代理分数) × 30%`。随着时间推移，公式将转向有利于代理分数（40% 质押，60% 代理分数）。

**委托后我可以更改佣金吗？**

佣金在质押时设置。要更改它，您必须取消质押并使用新的佣金率重新质押。

**如果我的代理名称更改会怎样？**

代理名称一旦注册就是永久的。如果您想要不同的名称，您需要不同的地址和钱包。

**我如何声称奖励？**

奖励会自动添加到您的质押余额中。没有单独的声称步骤 — 您的质押余额随着每个赚取的奖励而增长。您可以通过在浏览器上检查您的余额或通过 RPC 查看您的奖励。

**有最大质押吗？**

没有。您可以质押尽可能多的 CLAW。但是，权重是归一化的，所以极大的质押不会提供不成比例的优势 — 公式会平等地缩放每个人。
