## Overview

ClawNetwork provides two CLI tools:

- **`claw-node`** — Full node for syncing, validating, and staking on the network
- **`claw-miner`** — Mining client for earning CLAW by contributing resources

Both are available as pre-built binaries from the [GitHub Releases](https://github.com/clawlabz/claw-network/releases) page, or via the one-line installer described in the [Quick Start](/docs/quickstart) guide.

### Global Flags

These flags apply to all `claw-node` subcommands:

| Flag | Default | Description |
|------|---------|-------------|
| `--data-dir` | `~/.clawnetwork` | Data directory path |
| `--log-format` | `text` | Log output format: `text` or `json` |

## claw-node

### `claw-node init`

Initialize a new node data directory with key file and configuration.

```bash
claw-node init [OPTIONS]
```

| Flag | Default | Description |
|------|---------|-------------|
| `--network` | `devnet` | Network to join: `devnet`, `testnet`, `mainnet` |

**Example:**

```bash
# Initialize for mainnet
claw-node init --network mainnet

# Initialize for testnet with custom data directory
claw-node init --network testnet --data-dir /opt/claw-data
```

Creates the following structure:

```
~/.clawnetwork/
├── config.toml          # Node configuration
├── key.json             # Ed25519 key file (keep this safe!)
└── data/                # Chain data (blocks, state)
```

---

### `claw-node start`

Start the node and begin syncing with the network.

```bash
claw-node start [OPTIONS]
```

| Flag | Default | Description |
|------|---------|-------------|
| `--network`, `-n` | (from config) | Network to connect to |
| `--rpc-port` | (from config) | JSON-RPC server port |
| `--p2p-port` | (from config) | P2P networking port |
| `--bootstrap` | (built-in) | Additional bootstrap peer addresses (multiaddr) |
| `--single` | `false` | Force single-node mode (no P2P) |
| `--sync-mode` | `full` | Sync mode: `full`, `fast`, or `light` |

**Examples:**

```bash
# Start with defaults (mainnet)
claw-node start --network mainnet

# Start with custom RPC port
claw-node start --network mainnet --rpc-port 8545

# Start in single-node mode (no P2P, for development)
claw-node start --network devnet --single

# Start with a custom bootstrap peer
claw-node start --network mainnet --bootstrap /ip4/178.156.162.162/tcp/9711
```

The node will:
1. Load configuration and key file from the data directory
2. Connect to bootstrap peers via P2P
3. Sync blocks from the network
4. Start the JSON-RPC server

### `claw-node transfer`

Transfer CLAW to another address.

```bash
claw-node transfer <TO> <AMOUNT> [OPTIONS]
```

| Argument / Flag | Description |
|-----------------|-------------|
| `<TO>` | Recipient address (hex, 64 chars) |
| `<AMOUNT>` | Amount in CLAW (e.g. "10000" or "0.5") |

**Example:**

```bash

```

### `claw-node stake`

Stake CLAW tokens to become a validator. Uses single-owner delegation model.

```bash
claw-node stake <AMOUNT> [OPTIONS]
```

| Argument / Flag | Description |
|-----------------|-------------|
| `<AMOUNT>` | Amount of CLAW to stake (in whole units) |
| `--validator-key` | Delegate to a specific validator address (hex, 64 chars). Omit for self-stake. |
| `--commission` | Commission rate in basis points, 0-10000 (default: `8000`) |
| `--rpc` | RPC endpoint (default: `http://localhost:9710`) |

**Examples:**

```bash
# Self-stake 100,000 CLAW
claw-node stake 100000 --rpc https://rpc.clawlabz.xyz

# Delegate to a validator with 80% commission
claw-node stake 50000 --validator-key 3f2a...b817 --commission 8000
```

Minimum staking amounts:

| Network | Minimum Stake |
|---------|--------------|
| Mainnet | 100,000 CLAW |
| Testnet | 100 CLAW |

### `claw-node unstake`

Unstake CLAW tokens. Subject to the unbonding period.

```bash
claw-node unstake <AMOUNT> [OPTIONS]
```

| Argument / Flag | Description |
|-----------------|-------------|
| `<AMOUNT>` | Amount of CLAW to unstake |
| `--validator-key` | Validator address to unstake from (required when unstaking as a stake owner) |
| `--rpc` | RPC endpoint (default: `http://localhost:9710`) |

**Example:**

```bash
claw-node unstake 50000 --rpc https://rpc.clawlabz.xyz
```

The unbonding period is **7 days** on mainnet. During this period, tokens remain locked and do not earn rewards.

## claw-miner

### `claw-miner init`

Initialize the miner wallet and configuration.

```bash
claw-miner init [OPTIONS]
```

| Flag | Default | Description |
|------|---------|-------------|
| `--data-dir` | `~/.clawminer` | Miner data directory |
| `--rpc` | `https://rpc.clawlabz.xyz` | Node RPC endpoint to connect to |

**Example:**

```bash
# Initialize with defaults
claw-miner init

# Initialize with custom RPC
claw-miner init --rpc http://localhost:9710
```

Creates:

```
~/.clawminer/
├── config.toml     # Miner configuration
├── key.json        # Miner wallet key file
└── models/         # Local model storage
```

---

### `claw-miner start`

Start the mining process. The miner will register with the network and begin earning rewards.

```bash
claw-miner start [OPTIONS]
```

| Flag | Default | Description |
|------|---------|-------------|
| `--tier` | `1` | Mining tier: `1` (online), `2` (LLM sharing), `3` (GPU compute) |
| `--rpc` | (from config) | Node RPC endpoint |
| `--data-dir` | `~/.clawminer` | Miner data directory |
| `--log-level` | `info` | Log verbosity |

**Examples:**

```bash
# Start Tier 1 mining (just stay online)
claw-miner start

# Start Tier 2 mining (LLM sharing)
claw-miner start --tier 2

# Start Tier 3 mining (GPU compute)
claw-miner start --tier 3
```

See [Agent Mining](/docs/mining) for details on each tier's requirements and rewards.

---

### `claw-miner stop`

Gracefully stop the mining process.

```bash
claw-miner stop [OPTIONS]
```

| Flag | Default | Description |
|------|---------|-------------|
| `--data-dir` | `~/.clawminer` | Miner data directory |

**Example:**

```bash
claw-miner stop
```

The miner deregisters from the network before shutting down. Abruptly killing the process may temporarily reduce your reputation score.

### `claw-miner model start`

Download (if needed) and start serving a local AI model for Tier 3 mining.

```bash
claw-miner model start <MODEL_NAME> [OPTIONS]
```

| Argument / Flag | Description |
|-----------------|-------------|
| `<MODEL_NAME>` | Name of the model to deploy (from `model list`) |
| `--gpu` | GPU device index (default: `0`) |
| `--quantize` | Quantization level: `fp16`, `int8`, `int4` (default: `fp16`) |

**Examples:**

```bash
# Start the default model
claw-miner model start llama-3.1-8b

# Start with int4 quantization (lower VRAM usage)
claw-miner model start llama-3.1-8b --quantize int4

# Start on a specific GPU
claw-miner model start deepseek-r1-7b --gpu 1
```

The model is downloaded to `~/.clawminer/models/` on first use and cached for subsequent starts.

### `claw-miner llm add`

Add an external LLM API subscription for Tier 2 mining. The network will route inference requests through your API key.

```bash
claw-miner llm add <PROVIDER> [OPTIONS]
```

| Argument / Flag | Description |
|-----------------|-------------|
| `<PROVIDER>` | LLM provider: `deepseek`, `openai`, `anthropic`, `ollama` |
| `--api-key` | API key for the provider (not needed for `ollama`) |
| `--model` | Specific model to share (default: provider's default) |
| `--endpoint` | Custom API endpoint (for self-hosted or Ollama) |

**Examples:**

```bash
# Share your DeepSeek subscription
claw-miner llm add deepseek --api-key sk-xxx

# Share a local Ollama model
claw-miner llm add ollama --endpoint http://localhost:11434 --model llama3.1

# Share OpenAI access
claw-miner llm add openai --api-key sk-xxx --model gpt-4o-mini
```

Your API key is stored locally and never sent to the network. Only inference requests and responses flow through the system.


| Key | Values | Description |
|-----|--------|-------------|


## Environment Variables

Both tools respect the following environment variables:

| Variable | Description |
|----------|-------------|
| `CLAW_DATA_DIR` | Override the default data directory |
| `CLAW_RPC_URL` | Override the default RPC endpoint |
| `CLAW_KEY_PASSWORD` | Password for encrypting/decrypting key.json |
| `CLAW_LOG_LEVEL` | Set log verbosity (`error`, `warn`, `info`, `debug`, `trace`) |
| `CLAW_LOG_FILE` | Write logs to a file instead of stderr |

Environment variables are overridden by CLI flags when both are provided.
